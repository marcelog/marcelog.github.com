Starting WebRTC2SIP as a service without screen or console.

Introduction

Webrtc2sip is a much needed tool nowadays to enable webrtc in your web applications and sites and used with Asterisk and/or FreeSWITCH mostly.

Unfortunately this handy tool can't be started as a service as it is, since it will always start and read from standard input for commands (as of today, the only command available seems to be quit).

A lot of folks have resorted to use screen to start webrtc2sip as a service. This solution might for some people, but it is a kludgy way to get around this.

So I've made this simple patch available as a gist that adds a new option to webrtc2sip that can be used at startup: --without-console.

To apply the patch just download the gist to your local copy of the webrtc2sip svn repo: 

svn co https://webrtc2sip.googlecode.com/svn/trunk/ webrtc2sip
cd webrtc2sip
wget https://gist.githubusercontent.com/marcelog/47ef364b43c67f9f359d/raw/c80727107d0f468d878329d97a50b114463e8331/webrtc2sip_optional_console.diff
patch -p0 < webrtc2sip_optional_console.diff

Then continue building as usual: 

export PREFIX=/opt/webrtc2sip
./autogen.sh
LDFLAGS=-ldl ./configure --prefix=$PREFIX
make
make install

After that you can start webrtc2sip normally with console: 

webrtc2sip --config=/etc/webrtc2sip/config.xml

Or (by specifying the new option), without a console, like: 

webrtc2sip --config=/etc/webrtc2sip/config.xml --without-console

Enjoy!
